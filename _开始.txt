💡 核心知识速览
═══════════════════════════════════════════════════════════════════════════

【什么是 Rez？】
  ✓ 包管理系统（类似 npm, pip）
  ✓ 环境管理工具（创建隔离环境）
  ✓ 依赖解析器（自动解决版本冲突）
  ✓ 专为 VFX/游戏开发设计

【为什么用 Rez？】
  ✓ 管理复杂的依赖关系
  ✓ 避免版本冲突
  ✓ 为不同项目创建隔离环境
  ✓ 自动化环境配置

【基本概念】
  Package   → 自包含的软件单元 (例: myapp-1.0.0)
  Requires  → 依赖项 (例: ["python-3.9+", "maya-2022"])
  Variants  → 包的变体 (例: [["python-3.9"], ["python-3.11"]])
  Context   → 已解析的环境 (包含所有包和配置)
  Commands  → 加载时执行的配置 (设置环境变量)

【package.py 是什么？】
  • 包的配置文件
  • 定义包的元数据 (名称、版本、作者等)
  • 定义依赖项
  • 定义环境变量配置
  • 定义包的变体

  from rez import packages
  pkg = packages.get_package('myapp', '1.0.0')
  print(f'{pkg.name} v{pkg.version}: {pkg.description}')
  "

【验证包配置】

  # 如果没有错误输出，表示包配置正确

【配置版本约束】

      "python-3.9+<4",        # Python 3.9 系列
      "maya-2022+",           # Maya 2022 及以上
      "perforce-2021.1,2021.2"  # 特定版本

问题1: 包找不到

  ✓ 检查: package.py 文件是否存在

  ✓ 检查: 文件编码是否为 UTF-8
  ✓ 检查: 缩进是否正确（使用 4 个空格）
  ✓ 参考: 快速参考.md 中的示例

问题3: 版本约束错误
  ✓ 检查: 版本格式是否正确
  ✓ 参考: 快速参考.md 的版本约束语法部分

  ✓ 检查: commands() 函数是否正确定义
